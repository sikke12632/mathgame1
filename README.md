# 도형 게임

초등학교 5학년 1학기 6단원 `다각형의 둘레와 넓이`를 연습하는 2인용 정적 웹게임입니다. 기본 모드인 `도형 영토전`에 더해, 5문제를 풀어 스탯을 투자하고 가위바위보 전투를 하는 `도형 RPG 대전` 모드를 함께 제공합니다.

현재 버전은 기존 로컬 2인 모드에 더해 Firebase Realtime Database를 이용한 온라인 1:1 방 만들기/입장을 지원합니다. GitHub Pages에 정적 파일 그대로 배포할 수 있습니다.

## 실행 방법

별도 설치 없이 `index.html`을 Chrome에서 열면 실행됩니다.

```txt
outputs/shape-territory-war/index.html
```

로컬 서버로 확인하고 싶다면 이 폴더에서 다음 명령을 사용할 수 있습니다.

```bash
python -m http.server 5173
```

그 뒤 브라우저에서 `http://localhost:5173`을 엽니다.

## 빌드 방법

순수 HTML/CSS/JavaScript 정적 웹앱이므로 별도 빌드가 없습니다. `index.html`, `styles.css`, `gameCore.js`, `app.js`를 그대로 배포하면 됩니다.

온라인 배포에는 `firebase-config.js`도 함께 필요합니다. 자세한 순서는 [ONLINE_SETUP.md](./ONLINE_SETUP.md)를 참고하세요.

## 게임 규칙 요약

첫 화면에서 대전 형식을 고릅니다.

- `도형 영토전`: 문제를 풀어 얻은 칸으로 영토를 넓힙니다.
- `도형 RPG 대전`: 같은 문제 5개를 풀어 스탯 포인트를 얻고, 체력/공격력/방어력을 투자한 뒤 5턴 전투를 합니다.
- `혼자서 하기`: 플레이어 A가 학생, 플레이어 B가 규칙 기반 AI로 진행됩니다.
- `온라인 방 만들기`: 기존처럼 학생 2명이 방 코드로 함께 진행합니다.

### 도형 영토전

- 게임판은 항상 20×20, 총 400칸입니다.
- 플레이어 A는 왼쪽 위 2×2, 플레이어 B는 오른쪽 아래 2×2에서 시작합니다.
- 난이도에 따라 시작 시 검은 방해칸이 생성됩니다: 쉬움 10개, 보통 20개, 어려움 30개.
- 총 5라운드를 진행합니다.
- 매 라운드 두 플레이어가 같은 문제를 동시에 풉니다.
- 둘 다 정답이면 더 빨리 제출한 플레이어가 먼저 행동하고, 상대가 이어서 행동합니다.
- 같은 플레이어가 2회 이상 연속으로 먼저 행동했다면, 둘 다 정답인 라운드에서는 보호 규칙으로 상대가 먼저 행동합니다.
- 한 명만 정답이면 정답자만 행동하고, 둘 다 오답이면 행동 없이 넘어갑니다.
- 문제를 맞히고 행동할 수 있으면 정답 숫자만큼 칸을 획득합니다.
- 빈칸은 자기 영토와 상하좌우로 연결되어야 칠할 수 있습니다.
- 검은 칸은 칠할 수 없고, 연결 판정에도 사용할 수 없습니다.
- 상대 칸은 획득 칸의 절반까지만 중립화할 수 있으며, 중립화된 칸은 검은 칸이 됩니다.
- 쓰지 못한 획득 칸은 점수로 저장되지 않고 사라집니다.
- 남은 에너지가 있는 상태에서 색칠을 확정하면 확인 팝업이 뜹니다.
- 색칠 단계 제한 시간은 20초입니다. 시간이 끝나면 현재 선택한 칸만 자동 확정됩니다.
- 최종 점수는 `현재 자기 칸 수`입니다.

### 도형 RPG 대전

- 두 플레이어가 같은 문제 5개를 풉니다.
- 정답이면 그 문제의 정답 숫자만큼 스탯 포인트를 얻고, 오답이면 0점입니다.
- 체력은 투자 포인트 1점당 2씩 오릅니다.
- 공격력과 방어력은 투자 포인트 1점당 1씩 오릅니다.
- 공격력과 방어력은 같을 수 없습니다.
- 0점이면 공격형 또는 방어형 보정 1점을 선택할 수 있습니다.
- 전투는 5턴이며 가위는 공격, 보는 회복, 바위는 캐릭터 성향에 따라 공격 또는 회복입니다.
- 체력이 0이 되면 즉시 종료되고, 5턴이 끝나면 남은 체력, 문제 점수, 무승부 순으로 판정합니다.

## 문제 생성 규칙 요약

- 도형은 직사각형, 정사각형, 평행사변형, 삼각형, 마름모, 사다리꼴을 지원합니다.
- 학습 범위는 둘레만, 넓이만, 둘레 + 넓이 중 선택할 수 있습니다.
- 모든 문제는 템플릿 기반으로 생성됩니다.
- 쉬움은 필요한 길이를 직접 줍니다.
- 보통은 필요한 값 하나를 간단한 관계로 찾게 합니다.
- 어려움은 둘레, 배수, 절반, 필요 없는 정보 등을 이용해 먼저 필요한 값을 찾게 합니다.
- 삼각형, 마름모, 사다리꼴 넓이는 자연수 답만 나오도록 생성합니다.
- 문제 카드에는 이번 문제가 둘레인지 넓이인지 크게 표시합니다.
- 문제 유형과 도형은 선택 범위 안에서 한쪽으로 지나치게 몰리지 않도록 생성합니다.

## 현재 구현하지 않는 것

- 로그인
- 교사용 관리 화면
- 학급 랭킹
- 보너스 칸과 배경 효과

## v2 온라인 대전 확장 방향

기본 로컬 대전은 유지하면서, 간단한 온라인 방 만들기/입장 기능을 추가했습니다. Firebase 설정값을 넣고 GitHub Pages에 배포하면 학생들이 각자 기기에서 같은 방 코드로 접속해 플레이할 수 있습니다.

추가 후보 기능:

- 교사용 관리 화면
- 학급 랭킹
- 접속 끊김 안내
- 방 자동 정리
- 로그인 또는 익명 인증 기반 보안 규칙

확장을 위해 `gameCore.js`는 `GameState`를 하나의 직렬화 가능한 객체로 관리하고, `generateProblem`, `submitAnswer`, `selectCells`, `confirmPainting`, `nextTurn`, `calculateScore` 같은 순수 함수형 액션을 중심으로 구성했습니다. v2에서는 이 상태 객체를 서버나 실시간 채널로 동기화하는 구조를 붙이면 됩니다.

## 테스트

Node.js가 설치되어 있으면 다음 명령으로 핵심 규칙 테스트를 실행할 수 있습니다.

```bash
node tests/gameCore.test.js
```

테스트 범위:

- 시작 위치와 방해칸 생성
- 선택한 도형과 문제 유형만 출제되는지
- 넓이 문제가 자연수 답을 만드는지
- 삼각형 둘레 문제가 삼각형 조건을 만족하는지
- 검은 칸이 연결 통로로 쓰이지 않는지
- 연결되지 않은 빈칸이 자동 제외되는지
- 상대 칸 중립화 제한이 적용되는지
- 남은 획득 칸이 추가 점수로 저장되지 않는지
- 선택한 칸을 다시 클릭해도 기존 선택이 지워지지 않는지
- 동시 제출 뒤 선공/후공이 올바르게 결정되는지
- 보호 규칙, 단독 정답, 둘 다 오답 처리가 올바른지
- RPG 점수 합산, 스탯 검증, 전투 효과, 승패 판정이 올바른지
